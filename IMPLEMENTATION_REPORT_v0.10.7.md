# Scholia Studio v0.10.7 实现报告

本轮针对用户反馈的 7 个界面/交互问题做了集中修复，目标是减少默认打扰、提高多文件切换效率，并让参考文献管理进入可完整使用的状态。

## 已实现

1. 大纲面板稳定性
   - 修复大纲面板 CSS grid 行定义错误，避免“搜索/层级”和大纲树互相覆盖。
   - 搜索、层级筛选、展开、折叠改为紧凑单行工具条，减少刚打开时占用空间。

2. 参考文献面板
   - 移除表格/卡片切换控件，默认且只保留卡片视图。
   - 新增内嵌 BibTeX 表单，支持 article、inproceedings、book、phdthesis、mastersthesis、misc。
   - 按类型提示/补齐必填项：key、title、author、year、journal/booktitle/publisher 等。
   - 支持编辑已有条目，并写回原 `.bib` 文件。

3. 参考文献跳转
   - 点击参考文献卡片跳转到 `.bib` 时，不再关闭参考文献侧边栏。
   - 跳转时清理 Bib 预览悬浮卡片，不再出现“侧边栏 + 悬浮卡片”同时打开。

4. 设置页 PDF / LaTeX 组织
   - 设置页标签改为“PDF/LaTeX”。
   - LaTeX 构建区从“环境”移动到“PDF/LaTeX”下面。
   - 增加清理辅助文件快捷键：`Ctrl/Cmd+Alt+K`。

5. 类 VS Code 文件 Tab
   - 编辑区顶部新增已打开文件标签栏。
   - 支持快速切换已打开文件、关闭文件、查看未保存状态。
   - 每个标签提供“⇣”按钮，可把文件放入下方编辑组。

6. Markdown / TeX 默认不打开预览
   - 新打开 `.md`、`.tex`、`.bib`、`.txt` 默认进入编辑器模式。
   - 图片和 PDF 仍保持预览模式。
   - 用户手动打开预览后，普通 Tab 切换不再强制关闭用户已开启的预览。

7. 上下分栏编辑
   - 支持把已打开文件放入下方编辑组，形成上下分栏。
   - 支持拖动分隔条调整上下编辑组高度。
   - 支持拖动文件标签到编辑区底部创建下方编辑组。
   - 下方编辑组可以关闭，但不会关闭文件标签。

## 说明

本次的上下分栏实现为“编辑器组”级别的轻量实现，已经支持上下两个组、标签拖放、分隔条调整和独立编辑。后续如果继续做更接近 VS Code 的完整工作台，可以进一步扩展为：任意数量编辑组、左右/上下自由布局、文件在组间拖放排序、每组独立历史栈。
